## 🤔 고민한 점

### 아키텍처에 관한 고민
MVC와 MVVM 중 어느 아키텍처로 구현할 지 고민을 많이 했습니다. 
MVC의 단점이 뷰와 뷰컨의 결합도가 높고, 뷰컨이 방대해진다는 것이지만, 규모가 작은 프로젝트에서는 오히려 MVVM으로 구현하는 것이 오버엔지니어링이라 생각하여 MVC 패턴으로 결정했습니다.
하지만 학습의 일환으로 MVC에서 MVVM으로 추후에 리팩토링할 계획이 있습니다.
